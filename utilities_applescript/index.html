Returns the most appropriate translation of the specified string according to the localization of the specified application. 

on appl10n(
    key,
    appName,
    tableName)
Parameters
key

the string to localize, see Localizable.strings within the application bundle for available values and translations

appName

the name of the application from which to collect a localization

tableName

the name of the strings file from which to load the translation without the extension. For example, use "Localizable" to reference Example.app/Contents/Resources/en.lproj/Localizable.strings

Return Value

The localized string for the given key, or the key itself if no localization was found.

l10n

Returns the most appropriate translation of the specified string according to the workflow's bundle. 

on l10n(
    key)
Parameters
key

the string to localize, see Localizable.strings within this workflow for available values

Return Value

The localized string for the given key, or missing value if a value was not found for the key

Discussion

This workflow uses a simple hierarchical structure for localization to keep the keys organized and consistent. This means that the keys are not suitable for display as with simple "Foo" = "Foo"; translation by word or phrase. As a result, if the key is not localized, the handler will return missing value.

    theList)
Parameters
theList

A list of simple comparable values (string, number, date)

Discussion

Internally, this uses quickSortWithKeyHandler with the default key handler that causes the values of the list to be sorted directly.

quickSortWithKeyHandler

quickSortWithKeyHandler

An implementation of the quick sort algorithm that allows a key to be extracted from complex data types with a handler. 

on quickSortWithKeyHandler(
    theList,
    keyHandler)
Parameters
theList

A list of simple comparable values (string, number, date) or complex types such as records and nested lists. The latter must be accompanied by a keyHandler to extract a comparable value for sorting.

keyHandler

A handler that when given a value from theList returns the value on which that item should be sorted. If keyHandler is missing value then the values in the list will be sorted directly.

Discussion

This implementation of quick sort selects a starting pivot in the middle of the list to ensure optimal sorting of lists that are already partially sorted. Values are sorted in ascending order. Consider the following example: 

    set fruits to {{name:"Kumquat", color:"orange"},{name:"Apple", color:"red"},{name:"Lychee", color:"pink"}}

    on getFruitName(fruitInfo)
	    return name of fruitInfo
    end getFruitName

    set fruits to quickSortWithKeyHandler(fruits, getFruitName)

Acknowledgements

This handler is based on code for the quick sort algorithm provided by Kevin Bradley at MacScripter.

sendNotification

Displays a notification in Notification Center. 

on sendNotification(
    key)
Parameters
key

a Messages/ localization key as specified in Localizable.strings

Discussion

Loads Title, Message, and Details text from the workflow's localization and uses the strings to display a Notification Center notification to the user. Any combination of those subkeys may be specified in the localization.
